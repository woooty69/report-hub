How Claude is Creating a New Generation of Millionaires

요약

핵심 메시지

  • AI(특히 Claude)로 코딩을 전혀 못 하는 사람들이 실제 회사를 만들어 새로운 백만장자 세대가 생기고 있다는 주장. 진행자(Nate Herk)도 자신의 사업을 매일 Claude로 운영.
  • 지금이 **기회의 창(window)**이며 빠르게 닫히고 있으니, 진입 장벽이 낮은 지금 시작하라는 것이 영상의 취지.

Claude / Anthropic 배경 (진행자 인용 수치)

  • Claude는 Anthropic이 만든 AI. 표면적으론 ChatGPT 경쟁자처럼 보이지만:
    • 2주 전 단일 라운드에서 650억 달러 조달, 기업가치 9,650억 달러(1조 달러에 근접)로 평가 — 사상 처음 OpenAI를 추월.
    • 매출 런레이트가 470억 달러 도달(2024년 말 10억 달러 → 약 1년 반 만에 47배). 런레이트 = 기업들이 연 470억 달러 페이스로 지불 중이라는 의미.
  • “지구상 가장 똑똑하고 부유한 투자자들이 이 한 회사에 그만한 돈을 베팅했다”는 근거로 제시.

Vulcan 사례 — 코딩 못 하는 팀이 주(州) 계약을 따다

  • Vulcan: 정부 기관 내부에서 돌아가는 소프트웨어를 만드는 회사. 겉보기엔 엔지니어 100명의 작품 같지만, 팀 대부분이 코드를 한 줄도 못 씀 — 원하는 것을 Claude에 설명해서 회사 전체를 구축.
  • 창업자 3명 중 2명은 코딩 불가, 그중 Tanner Jones는 고교 자바스크립트 수업 이후 코드를 만진 적 없음. 첫 프로토타입은 Claude Code가 나오기 전이라 일반 Claude 앱에서 코드를 복붙하며 만듦.
  • 그 복붙 프로토타입이 실제 제품이 되어 버지니아 주 계약을 기존 컨설팅펌 견적의 약 10% 가격에 수주. 이후 주지사가 모든 주 기관에 해당 AI 규제 검토 도입을 의무화하는 행정명령 서명 → “버지니아가 법으로 우리 제품을 요구”, 연간 10억 달러 이상 절감 주장.
  • Tanner의 정리: “언어를 이해하고 비판적 사고를 할 수 있으면 Claude Code를 잘 쓸 수 있다.” 즉 새 스킬은 원하는 것을 정확히 설명하고, 돌아온 결과를 비판적으로 사고하는 것.

Claude Code가 다른 이유 — 4가지

  1. 일을 실제로 한다(It does the work): 평이한 영어로 원하는 것을 설명하면 그대로 구축 — 한 문장에서 앱 하나, 웹사이트, 반복 작업 자동화를 몇 달이 아니라 몇 분에.
  2. 에이전틱(Agentic): 질문에 답만 하는 챗봇과 달리 목표를 향해 스스로 행동 — 만들고 → 테스트하고 → 버그를 찾고 → 고침. 중간 과정을 사람이 안 해도 됨.
  3. 병렬 작업(Parallel): 여러 Claude 에이전트를 동시에, 자는 동안에도 돌릴 수 있음.
  4. 기억한다(Memory): 내 사업·팀·우선순위·과거 실패를 기억. 진행자는 “나보다 기억력이 좋아서 할 일을 상기시켜 준다”고 표현.
  • 이 4가지가 합쳐지면 아이디어와 실제 제품 사이의 간극이 사라짐.

시장 신호와 ‘기회의 창’

  • Y Combinator(세계 최고 스타트업 액셀러레이터, Airbnb·Stripe·DoorDash 배출) 최신 배치에서 절반 이상의 스타트업이 Claude로 개발 — 배치 내 가장 많이 쓰인 AI. 1년 전엔 OpenAI가 90%+였음. “지금 창업자들은 어떤 도구가 최고인지 논쟁하지 않고 이미 Claude를 골랐다.”
  • 일반 직장·소상공인에도 같은 격차 — Claude로 반복 업무를 처리하는 사람이 더 많이 해내고 더 큰 기회를 받음. “AI가 당신의 일자리를 빼앗는 게 아니라, AI를 쓸 줄 아는 사람이 빼앗는다.”
  • 초기 인터넷 비유: 기본 HTML을 배워 사업을 온라인화하거나 페이스북 광고를 일찍 이해한 사람들이 값싼 관심(attention)을 선점해 10년간 복리 효과를 누림. “Claude로 이기는 사람들은 당신보다 똑똑한 게 아니라 먼저 시작했을 뿐.” 지금은 AI 네이티브가 진짜 이점이지만 2년 뒤엔 기본값이 됨.

지금 시작하는 법 — 4단계

  1. 유료 Claude 플랜 가입: Claude Code 접근에 필요. 입문 플랜 월 20달러로 시작 충분(추후 업그레이드 가능). “당신이 고용할 가장 저렴한 직원.”
  2. 진짜 과제 하나 선택(10개 아님, 딱 1개): 미루던 일, 반복 작업, 원래 프리랜서에게 외주 줄 만한 일(영수증 정리, 주간 보고서, 이메일 분류, 세금 등) 또는 메모앱에 쌓아둔 창업 아이디어. 그 하나가 첫 프로젝트.
  3. Claude Code에 그 과제를 최대한 명확히 설명(무엇인지, 지금 어떻게 하는지, 성공이 뭔지). 단, 바로 만들게 하지 말고 먼저 논쟁·브레인스토밍·악마의 변호인을 시킬 것 — AI 모델은 아첨(sycophantic) 경향이 있어 사용자를 기쁘게 하려 구멍 있는 아이디어도 띄워줌. 계획(planning)이 매우 중요.
    • 진행자의 ‘the roast’ 패턴: 서브에이전트 ‘위원회’를 띄워 한 명은 아이디어를 강하게 옹호, 한 명은 구멍을 지적(“네 아이디어는 별로”), 한 명은 객관적 근거·리서치. 메모리 덕에 위원들이 현재 사업 상황을 알고 판단. 모든 주장은 ‘바이브’가 아니라 실제 근거로 뒷받침. 마지막에 하나의 판정 — Go / Reshape / Kill. 살아남거나 재구성한 아이디어만 다음 단계로.
  4. 실제 구축(Build): 큰 프로젝트는 조각내어 가장 작은 동작 버전부터 시작해 키움. Claude가 “완료”라 해도 곧이곧대로 믿지 말고 실제 예시로 실행해 결과를 보여주게 함(검증, verification). “사람이 준 결과라면 뭘 확인하겠는가”를 그대로 AI에 시킴 — AI는 브라우저도 제어할 수 있어 사람이 컴퓨터로 하는 검증을 다 함.

반복 개선과 마무리

  • 첫 버전은 보통 60～70% 완성도 — 정상. 실패로 취급하지 말고 골든 데이터/피드백으로 취급. 좋았던 점·틀린 점을 평이한 영어로 알려주고 스스로 고치게 함 → 스킬과 메모리가 쌓이며 점점 좋아짐.
  • 이해 안 되는 게 있으면 설명을 요청 — “세상에서 가장 똑똑한 절친이자 멘토”처럼 다루면 학습됨. 한 과제가 되면 다음 과제로. “한 번에 하나씩” 이렇게 전 사업을 Claude로 운영하게 됨.
  • (홍보) 무료 스쿨 커뮤니티에 7일 챌린지, ‘roast’ 스킬 파일, ‘AI 운영체제’ 무료 강의 제공 — 코딩 몰라도 되고 명확히 사고할 줄만 알면 됨.
  • 결론: “다음 Vulcan을 이번 주말에 만들 필요는 없다. 월 20달러 구독과 과제 하나가 지금의 진입 장벽 전부이며, 그것만으로 세계 인구의 95%보다 앞서게 된다.”
