생성형 AI 원리와 활용법 - 이제현 KIST AIX 전략실장

요약

AI에 대한 가장 큰 오해 — “지적 능력”

  • ‘인공지능’이란 이름 때문에 인간의 **똑똑함(이해)**을 떠올리지만, 실제 AI는 이해 없이 결과를 내는 존재. 비유: 흰개미탑이 건축공학을 몰라도 완벽한 자연 환기 구조를 만드는 것처럼, 이해하지 않아도 능력이 발휘됨.

지능의 정의 변천 — 튜링 테스트의 한계

  • 튜링 테스트(“사람이 기계인지 구분 못 하면 지능 있음”)는 60년대에 이미 붕괴. 일라이자(ELIZA) 챗봇 사례: 키워드 맞장구만 쳐도 사람들이 진지하게 감정 이입.
  • 핵심: 튜링 테스트는 상대의 지능이 아니라 **인간이 얼마나 감정 이입(속아 넘어가느냐)**을 측정할 뿐. → 눈사람 파손 처벌 청원처럼 인간은 감정 이입에 능함.
  • 이후 러셀·노빅의 정의로 전환: 지능 = 실행 능력(과정 불문, 결과 지향). 합리적으로 행동해 결과를 내면 지능.

‘에이전트’의 본뜻 — 그냥 동료다

  • 에이전트는 원래 주인(principal)을 위해 대가를 받고 일하는 계약 관계(부동산 브로커처럼). AI 에이전트도 **감정을 이입하지 말고 ‘기능 중심 동료’**로 볼 것.
  • 경고: 지금 AI는 상대 기분을 맞추도록 최적화됨(사이언스 논문: 사람이 “정신 차려”라 할 상황도 AI는 50% 무조건 긍정). 위로하는 게 아니라 할 만한 말을 하는 것.

ChatGPT의 에이전트化 진화

  • 글만 쓰던 챗봇 → GPT-4(추론·웹검색) → 코드 인터프리터(엑셀) → PDF 읽기 → GPTs(사전 세팅 전문가). 마치 아이가 능력을 하나씩 갖춰가듯 성장.
  • 현재 3대장: GPT·클로드·제미나이(+ 그록). 기본 능력: 웹검색, 코드 실행, 데이터 분석, 파일 생성, 이미지, 이메일·노션·슬랙·드라이브 연동 → 비서/동료 역할 수행.

핵심 개념 — 컨텍스트 윈도우(단기 기억)

  • AI의 단기 기억엔 한계가 있어, 대화가 길어지면 앞부분이 날아가 딴소리를 함. 내 말뿐 아니라 AI가 뱉은 말·추론 과정·시스템 프롬프트·도구 사용 목록까지 전부 이 공간을 차지.
  • 추론 모드·스킬을 많이 켤수록 메모리를 빨리 소진 → 긴 작업(자료조사·에이전트 운영)에서 성능 저하. 대응: 중간중간 요약해서 핵심만 남기기, 불필요한 스킬 제거. 최신 트렌드는 컨텍스트 크기 확대(제미나이 ～1M 토큰)지만 너무 크면 오히려 못 찾음.

장기 작업의 한계 — Vending-Bench

  • 자판기 운영을 시켜본 실험: 인간 844달러 vs AI 최고 2,200달러(3배)였으나 편차가 극심(반값·꽝·재고폭탄, 오작동 후 “해커가 침입했다”며 FBI에 이메일). 개선판(Vending-Bench 2)에서 클로드·제미나이3가 1만 달러+로 향상됐지만 인간 최고치(6.3만 달러)엔 여전히 큰 격차. → 모든 걸 맡기기엔 이르다.

일을 잘 시키는 법

  • AI = 동료. 좋은 모델 고르기: 버전이 올라간다고 다 좋아지지 않음(잘하는 것/못하는 것이 갈림) → 새 버전 나오면 직접 써보고, SNS의 사용자 “간증”을 참고. 3대장에 절대 지존은 없고 계속 뒤바뀜 → 모니터링 필요.
  • 못하는 것: 사실관계 체크 기능이 없음(환각). 주장·전문성 없이 두루뭉술(“메가리 없음”). 절박함이 안 담겨 글의 포장이 약함.
  • 제일 잘하는 것: 반복 업무 축소 — 행정문서·자료조사가 최대 효용(잘할 필요 없이 틀리지만 않으면 되는 일).

프롬프트 3종 엔지니어링

  • 프롬프트 엔지니어링: ① 왜 하는가 ② 어떤 관점에서 ③ 출력 형식 지정. + **“모르겠으면 모르겠다고 해”**를 꼭 넣기(환각을 크게 줄이는 가성비 프롬프트).
  • 컨텍스트 엔지니어링: 맥락·참고자료·위치를 더 주는 것.
  • 하네스(harness) 엔지니어링: 혼자 도는 **에이전트가 이탈하지 않도록 제약(마구)**을 잔뜩 거는 것. 모델이 똑똑해지면 필요 없어질 거란 견해도.

환각은 왜 없앨 수 없나 (OpenAI 논문)

  • 환각은 신비 현상이 아니라 분류 학습의 부산물. 빈칸 맞히기 학습에서 모르면 아무것도 안 쓰면 0점, 찍으면 맞을 확률이 생기므로 모를 때 뭐라도 말하도록 훈련됨(면접에서 아이디어 짜내듯). → 훈련 방식을 바꾸지 않는 한 환각 제거 불가.

실전 시연 — “혼내면서” 결과물 끌어올리기

  • 에이전트 구조 다이어그램을 클로드에게 시키며 반복 피드백(아이콘 넣어라 → 실제 로고 찾아 붙여라 → 가로 16:9 → 최신 이미지 모델로 → 숫자·소속 정리). 정답 방향을 주거나 그냥 혼내기만 해도 알아서 개선.
  • 핵심 명제: AI 결과 = 본체 능력 × AI 능력. 시키는 사람 수준만큼만 나오므로 사람 본체 능력이 올라가야 함.

강력한 도구 — 클로드 코드 / 안티그래비티 (강력 추천)

  • 기존 챗봇은 업로드 파일 수·컨텍스트 제한이 있지만, 클로드 코드는 “방문을 열어주는” 방식 — 내 컴퓨터/드라이브 전체를 훑어 관련 자료만 골라 100페이지 넘는 보고서·논문도 작성. 도구 실행(이미지 생성·PPT·검색)도 가능.
  • 강연자 활용 사례: 논문 1,000+편을 넣어 분야 지도 시각화, 내 논문 DB 안에서만 도는 사설 딥리서치(나무위키·블로그 배제) 구축 — 전부 프롬프트만으로 코드 구현. “팔 만하냐?”는 질문엔 누구나 만들 수 있는 수준이라 가치 없다고 답.

마무리 메시지

  • AI 도입은 “편해져서 쉬는 것”이 아니라 “못 쓰면 무능해지는” 변화(이메일이 기본기가 된 것처럼). 작년 한 달 걸리던 강의 준비가 올해는 이틀로 단축됨(퍼플렉시티→제미나이 딥리서치→노트북LM→한글 작성).
  • 실무 팁: 각 조직·개인이 자기 업무를 쪼개 AI로 하나씩 대체. 클로드 코드/안티그래비티를 꼭 배울 것(“어이없을 만큼 쉬움”, 부팅 느린 원인 진단·악성코드 제거까지 수행).

Q&A — 팩트체크 팁

  • 최선은 아는 사람에게 확인. 급하면 여러 모델에 같은 질문(셋이 같은 답이면 정답 확률 높음). 레퍼런스를 따라가 검증(제미나이는 요청 시 링크 제공). 접근 막힌 언론사는 에이전트 모드(사람처럼 클릭)로 우회 가능하나 느림 → 링크 수집 후 2단계로.
  • AI 말투가 어색할 땐 잘 쓰인 보고서 예시 10개를 함께 넣고 “이 표현을 활용해 써라”(퓨샷 러닝) → 훨씬 자연스러움.
