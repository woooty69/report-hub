I Tested Every AI for Trading Only 3 Are Worth Using

I Tested Every AI for Trading - Only 3 Are Worth Using

채널: Michael Automates | 업로드: 2026-05-15 | 영상 보기

1. 요약

영상 개요

  • 주요 AI 도구(Claude, OpenAI Codex, TradingView Remix, DeepSeek, xAI 등)를 트레이딩 용도로 전부 테스트하고, 용도별 최적 AI를 정리한 영상.
  • 다루는 유스케이스 5가지: ① 트레이드 셋업 발굴 ② 시장 전체 돌파(브레이크아웃) 스캔 ③ 전략 생성·백테스트 ④ 전략 파라미터 최적화 ⑤ 거래소 자동매매. 코딩 능력 불필요를 전제로 함.
  • 결론적으로 “쓸 만한 건 3개”: TradingView Remix(차트 분석·스캔·최적화), Claude/Codex + 백테스팅 엔진(전략 생성), Signum(자동매매 실행).

① 트레이드 셋업 발굴 — TradingView Remix

  • TradingView 자체 AI ‘Remix’(크롬 확장)가 최적. 차트에 셋업을 자동으로 그려주며, 시연 트레이드는 실제 수익으로 마감.
  • 사용자의 요금제·쿼터·현재 보고 있는 차트까지 인식하는 완전 통합형이라는 점이 강점.
  • 첫 시도에서 레벨 배치를 틀리는 등 오류가 있을 수 있으나, 대화하며 교정하는 방식으로 활용 가능.
  • Claude·Codex 등으로도 시도해 봤지만 TradingView에 직접 통합된 Remix의 정확도가 더 높았다는 평가.

② 시장 스캔(돌파 탐지) — Remix + Signum

  • Remix에 프롬프트를 주면 전체 DB에서 바닥권 형성 후 돌파한 코인을 찾아줌(시연에서 LINK, SANA 등 4개 제시, 자체 지표로 검증 시 실제 돌파 확인).
  • 다만 결과가 4개 수준으로 제한적. 진행자의 서비스 Signum은 수백 개 돌파를 탐지한다고 소개(Ondo, Zcash +113%, Memecore 사례). 주식·금속·원자재 확장 예정이라며 텔레그램 그룹 홍보.

③ 전략 생성·백테스트 — Claude(Opus 4.6)/Codex + 백테스팅 엔진

  • 파라미터·지표 조합을 수작업으로 실험하면 시간이 오래 걸리므로 AI에 위임.
  • Claude Opus 4.6 출시 이후 AI가 전략을 처음부터 끝까지 스스로 만드는 것이 가능해졌다고 평가. Codex도 가능.
  • 핵심은 AI 자체가 아니라 백테스팅 엔진: TradingView와 동일하게 백테스트할 수 있는 엔진이 없으면 AI가 제대로 검증하지 못하고 실수함.
  • 커스텀 지표를 주고 몇 시간씩 조합 실험을 시키는 방식도 소개. MACD·RSI·거래량은 누구나 쓰므로 독자적 지표로 차별화된 전략을 만들 것을 권장.

④ 파라미터 최적화 — Remix

  • 코드 접근 권한이 없는 전략도 Remix가 수많은 파라미터 조합을 테스트해 최적값 제시.
  • 시연: Gaussian Channel 전략을 BTC, ETH, SOL, BNB, DOGE, ADA에서 테스트 → 도지코인 기준 기본 파라미터 1.48% 수익(더 큰 낙폭) vs 최적화 후 977% 수익(MDD 82%), ADA 기준 48%/MDD 75% → 1,166%/MDD 52%로 개선.
  • 단, 최적화가 평범한 전략을 최고 전략으로 만들 수는 없음. 전략 자체의 품질과 과최적화(overfitting) 여부 확인이 선행돼야 함.

⑤ 자동매매 — TradingView 알림 + Signum 웹훅

  • 완성 전략은 TradingView 알림(웹훅)으로 자동화: 봇 생성 → 알림에 제목·메시지·웹훅 URL 복사 → 만료일 최대로 설정 → 전략 신호 발생 시 거래소에서 자동 매매.
  • 진행자 본인 포트폴리오도 이 방식으로 운영 중(현재 강세장이 아니라 대부분 비트코인 위주). 자체 유료 마스터클래스 홍보 포함.

AI 트레이딩 에이전트에 대한 경고

  • AI 에이전트에 포트폴리오를 통째로 맡기는 것에 반대. 이유: 백테스트가 없거나 검증 불가(블랙박스), 자기 차트에서 확인 불가, 과최적화 여부 판단 불가.
  • 심리적 문제도 큼: 전략을 이해하지 못하면 연속 손실 구간에서 그것이 정상 범위인지 알 수 없어 (설령 좋은 에이전트라도) 중도 포기하게 됨. 트레이딩의 50% 이상은 심리라는 지적.
  • 대안: 직접 전략을 만들어 이해하거나, 오래 검증된 전략이 포함된 시스템을 사용할 것. 어떤 시스템이든 손실 트레이드는 반드시 있으며, 그것이 정상 범위임을 알아야 버틸 수 있음.

유의점 (영상 자체 고지)

  • 영상 말미·설명란에 투자 조언이 아니며 암호화폐는 전액 손실 위험이 있다는 면책 고지. 진행자 본인 유료 서비스(Signum, AutoTrading Masterclass) 홍보가 포함된 영상임을 감안하고 볼 것.

2. 스크립트 (상세 정리)

원문 자막의 저작권 보호를 위해 발언 내용을 구간별로 상세히 재정리한 버전입니다. 원문 자막 전문은 영상의 ‘스크립트 보기’ 기능에서 확인할 수 있습니다.

인트로와 아젠다 (00:00～01:48)

어떤 AI가 트레이딩에 최적인지 전부 테스트했다며 영상 구성을 소개. 트레이드 셋업 발굴, 펌핑 전 코인 매수를 위한 돌파 스캔, 전략 생성·최적화·백테스트, 거래소 자동매매(24시간 무인 운영), 그리고 AI 에이전트에게 포트폴리오를 맡겨도 되는지까지 다룬다고 예고. 초보자와 고급 사용자 모두 적용 가능하고 코딩이 필요 없다는 점을 강조. 프롬프트와 도구 링크는 영상 후반과 설명란에 제공. 투자 조언이 아니며 암호화폐는 전액 손실 가능하다는 면책 고지.

트레이드 셋업 발굴: TradingView Remix (01:48～03:37)

차트 분석과 셋업 자동 작도에는 TradingView 자체 AI인 Remix가 최고라고 결론. 크롬 확장으로 브라우저에 통합되어 사용자 요금제, 남은 쿼터, 현재 보고 있는 차트를 모두 인식. 시연에서 프롬프트 하나로 전체 분석과 수익 트레이드를 얻었으나, 첫 시도에서 트레이딩 박스 레벨을 잘못 배치해 교정이 필요했음을 언급. 오류가 있어도 차트에 대해 AI와 대화한다는 개념으로 접근하면 유용하며, “차트 앞에 혼자가 아니게 된다”고 표현. Claude와 Codex로도 같은 작업을 시도했지만 TradingView에 직접 통합된 Remix의 정확도를 따라오지 못했다고 평가.

시장 스캔: 돌파 코인 찾기 (03:37～05:56)

“바닥 돌파 지점에서 사야지 이미 오른 뒤에 사면 안 된다”는 원칙 아래, Remix에 스캔 프롬프트를 주면 전체 차트·지표 DB에서 바닥권 형성 후 돌파한 코인을 찾아줌. 시연에서 4개 코인을 제시했고, LINK(“가장 깔끔한 돌파”)와 SANA를 본인이 만든 바닥권 탐지 지표로 검증한 결과 실제 돌파가 확인됨. 다만 4개로 제한적이라는 한계를 지적하며 자신의 서비스 Signum에서는 수백 개의 돌파를 찾을 수 있다고 소개. Ondo(돌파 후 펌핑·추세 전환), Zcash(돌파 이후 +113%), Memecore(무명 코인 돌파 후 급등) 사례를 제시. 주식·금속·원자재로 확장 예정이며 텔레그램 그룹 가입을 권유.

전략 생성·백테스트: Claude/Codex + 백테스팅 엔진 (05:56～08:08)

TradingView의 전략 리포트만으로는 파라미터 변경·지표 추가 등 수많은 조합을 비교하는 데 시간이 너무 걸리므로 AI에 맡기는 것이 답이라고 설명. Claude Opus 4.6 출시 시점 영상에서 다뤘던 방법이 최고의 솔루션이며, AI가 전략을 완전히 스스로 생성할 수 있게 됐다고 평가. 핵심은 특정 AI가 아니라 TradingView와 동일하게 백테스트하는 백테스팅 엔진이라고 강조 — 이것이 없으면 AI가 백테스트 방법을 몰라 실수를 한다는 것. Codex로도 가능하며 관련 영상을 각각 제작했다고 안내. 나아가 커스텀 지표(직접 제작 또는 TradingView 컬렉션 다운로드)를 주고 몇 시간 동안 새 전략 조합을 실험시키는 방법도 소개. MACD·RSI·거래량은 모두가 쓰므로 독자적인 지표로 유니크한 전략을 만들라고 권장.

파라미터 최적화: Remix (08:08～10:40)

코드 접근 권한이 없어도 Remix가 파라미터 조합을 대량 실험해 최적값을 제시. Gaussian Channel 전략을 BTC, ETH, SOL, BNB, DOGE, ADA에서 테스트해 “주요 암호화폐 전반에서 최적의 위험조정 균형”이라는 파라미터를 받은 시연 공개. 검증 결과 도지코인에서 기본 파라미터는 1.48% 수익에 더 큰 낙폭, 최적화 버전은 977% 수익(MDD 82%). ADA에서는 기본 48% 수익/MDD 75% 대비 최적화 1,166% 수익/MDD 52%로 개선. 어떤 변형이 포트폴리오에 더 적합한지 자동 판단하는 방법은 별도 영상(“TradingView has AI”)에서 다뤘다고 안내. 단, 옵티마이저도 평범한 전략을 최고로 만들 수는 없으므로 전략 자체의 품질과 과최적화 여부 확인이 우선이라고 강조. 본인 전략의 자산별 백테스트 결과를 나열하며 VIP 멤버에게 제공 중이라고 홍보.

자동매매: 알림 + Signum 웹훅 (10:40～13:43)

전략이 완성되면 TradingView 알림으로 자동화하는 절차를 시연. Signum에서 데모 봇 생성(BTC/USDC로 설정) 후, 전략 알림 생성 창에 봇의 제목·메시지·웹훅 URL을 복사해 넣고 반드시 ‘적용’을 클릭할 것, 만료일은 가능한 한 미래로(가능하면 무기한) 설정할 것을 당부. 이후 전략이 매매 신호를 낼 때마다 알림이 트리거되어 거래소에서 자동으로 매수·매도 실행. 본인 계정에는 비트코인 봇 여러 개와 소수의 알트코인 봇이 있으며, 지금은 강세장이 아니라 대부분 비트코인 위주라고 언급. 포트폴리오 구축·관리, 리스크 관리, 코인 선정, 전략 선택과 자금 배분을 다루는 유료 AutoTrading Masterclass를 소개.

AI 트레이딩 에이전트 비판 (13:43～16:05)

AI 에이전트가 포트폴리오 전체를 맡을 수는 있지만 본인은 반대한다는 입장. 근거: 에이전트는 대개 백테스트가 없고, 자기 차트에서 과거 성과를 직접 검증할 수 없으며(남이 주는 백테스트는 신뢰 불가), 원하는 자산에 적용해 과최적화 여부를 확인할 수도 없음. 또한 전략이 뭘 하는지 이해할 수 없으면 심리가 돈의 움직임과 동기화되지 않음 — 5연속 손실이 났을 때 그것이 정상 범위인지 알 수 없어, 설령 최고의 에이전트라도 중도 포기하고 나중에 후회하게 된다는 것. 트레이딩의 50% 이상은 심리이며, 손실 트레이드가 게임의 일부임을 이해해야 시스템을 오래 유지할 수 있다고 강조. 대안으로 직접 전략을 만들어 완전히 이해하거나, 오래 검증된 전략이 포함된 시스템을 쓰라고 권고. “AI가 포트폴리오를 자동화할 수는 있지만 반드시 백테스트가 필요하다”로 요약.

마무리 (16:05～끝)

용도별 최적 AI를 정리했으니 설명란에서 프롬프트(셋업 발굴, 돌파 스캔, 전략 최적화)와 백테스팅 엔진, Signum 링크를 확인하라고 안내. 더 깊이 다뤘으면 하는 주제를 댓글로 요청하며, 전략 구축에 관심 있다면 다음 영상을 보라고 권하고 마무리.
