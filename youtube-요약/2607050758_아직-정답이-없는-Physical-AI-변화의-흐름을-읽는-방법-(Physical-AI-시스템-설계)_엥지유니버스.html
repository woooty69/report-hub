아직 정답이 없는 Physical AI, 변화의 흐름을 읽는 방법

요약

문제의식 — 피지컬 AI는 ‘아직 정답이 없는’ 분야

  • 예시: 로봇에게 “바나나랑 같은 색 물체를 집어봐”라고 하면, 사람은 자연스레 이해하지만 로봇에겐 언어 이해 + 시각 인식 + 행동 계획 + 로봇 제어가 모두 연결된 복잡한 문제.
  • 기존 로봇은 정해진 환경·규칙에서는 정확하지만, 물체 위치·조명이 바뀌거나 예상 밖 명령이 오면 수많은 예외 상황 발생.
  • 핵심 어려움: 변화 속도가 너무 빠름. 오늘 최고 성능 모델이 몇 달 뒤 새 구조로 대체되고, 로봇 ‘성능’은 하나의 숫자로 비교 불가(로봇 종류·데이터·센서·행동 출력 주기에 따라 결과가 달라짐).

핵심 주장 — “최신 논문 이름 좇기보다 ‘왜 등장했는지’를 보라”

  • 새 기술 대부분은 앞선 방식이 못 푼 문제에 대한 답이고, 그 답이 또 새 문제를 드러내며 다음 연구로 이어짐. 따라서 결과만 보지 말고 문제→해결→새 한계의 연결을 봐야 함.

아직 수렴하지 않은 영역들 (경쟁 중인 선택지)

모델 아키텍처:

  • 하나의 거대 모델이 장면 이해·추론·행동 생성을 모두 담당 vs 계층형(상위: 복잡한 판단·장기 계획, 하위: 짧은 주기로 실제 움직임 생성).
  • 장면+언어→행동 직접 출력 vs “컵 쪽으로 팔 이동” 같은 언어 중간표현을 거쳐 세부 행동으로 변환 vs 미래 장면을 먼저 예측하고 그 상태 도달 행동을 생성.

행동 출력 형태:

  • 관절 위치·속도를 언어 토큰처럼 하나씩 생성(추론이 조금만 느려도 움직임이 끊김) vs 여러 스텝을 묶는 액션 청크(부드럽지만 환경 급변 시 수정 어려움) vs 디퓨전/플로우 매칭 생성모델로 연속 구간 전체 생성.
  • 생성모델을 쓰는 이유: 같은 컵을 짚어도 왼쪽/오른쪽 접근 등 행동이 하나로 정해지지 않는(다중 분포) 특성을 표현하기 위함.

데이터 확보:

  • 사람이 로봇 직접 조종 / 여러 기관·로봇 데이터 통합(오픈 로봇 데이터셋) / 성공뿐 아니라 실패·복구 과정 수집 / 인터넷 이미지·영상에서 일반 지식 학습 후 전이 / 사람 작업 영상 관찰.
  • 단, 인간과 로봇은 몸 구조가 달라 사람 영상엔 ‘관절 각도’ 정보가 없음 → 인간 행동을 로봇 좌표계·관절 명령으로 변환하는 별도 과정 필요.

가상 환경(시뮬레이션·합성 데이터):

  • 실제 공장·작업공간을 3D 스캔해 디지털 트윈으로 / 개발자가 직접 배치 / 생성형 AI로 가상 물체·장면·작업·초기조건·보상·성공판정까지 자동 생성.
  • 장점: 실제 로봇 1대보다 훨씬 많은 조건(위치·조명·마찰·초기자세)을 수천 개 동시 실행. 위험·희귀 상황도 반복 생성.

Sim-to-Real 문제

  • 시뮬레이터에서 잘 움직인 로봇이 실제에선 성능 저하 — 카메라·센서 노이즈, 모터 명령 지연·백래시(기어 유격), 마찰·질량·관성 값 불일치, 표면 거칠기·오염, 변형 재질, 미끄러짐 접촉 등은 정확히 모델링 어려움. 작은 차이가 제어 주기 동안 누적되면 실패.
  • 해법: 마찰·질량·센서 노이즈를 의도적으로 다양화하는 도메인 랜덤화, 정밀 디지털 트윈, 시뮬레이션 학습 모델을 소량 실제 데이터로 재조정, 배포 후 실패·예외를 다시 학습에 반영하는 지속 학습(continual learning).

배포·추론 속도

  • 텍스트 AI는 답이 수백ms 늦어도 되지만, 움직이는 로봇은 명령 지연이 성능뿐 아니라 안전에 직결(사람 근처·물체 파지 중).
  • 해법: 상위 추론은 외부 연산 장치, 빠른 행동 생성은 로봇 내부 경량 모델로 분리 / 모델 양자화·경량화로 온디바이스 실행.

발전의 흐름(계보) — 문제가 다음 구조를 낳았다

  1. LLM 플래닝: “식탁 위 물병 가져와” → 물병 찾기→이동→잡기→가져오기로 작업 분해. 강점은 복잡한 언어 명령의 단계 분해. 한계: ‘물병을 잡는다’ 계획과 실제 파지 움직임 사이 큰 간격(관절 궤적·힘 제어를 직접 못 만듦).
  2. VLA(Vision-Language-Action): 장면+명령+행동을 하나의 모델로 연결해 실제 실행 행동을 직접 출력. 한계: 로봇 움직임은 정답이 하나가 아니고, 시간에 따라 부드럽게 이어져야 하며 작은 오차가 누적됨.
  3. 액션 청크 + 생성 정책(디퓨전/플로우 매칭): 연속·다양한 움직임 표현. 한계: 짧은 구간 생성에 집중하면 작업 전체 목적 고려·계획 수정 능력 부족(예: 서랍 손잡이가 안 보이거나 물체가 없을 때).
  4. 계층형 시스템: 상위(느리지만 상황 해석·목표 결정) + 하위(짧은 주기 행동 생성)로 분리 → 빠른 행동과 복잡한 추론을 함께.
  • 데이터 쪽도 동일 흐름: 오픈 로봇 데이터 → 합성 데이터·시뮬레이션 → 디지털 트윈 → 지속 학습.

새 기술을 볼 때의 3가지 질문

  1. 기존 방식의 어떤 문제를 해결하려는가
  2. 그 문제를 어떤 아이디어로 풀었는가
  3. 무엇을 여전히 못 풀었는가
  • 이 셋을 보면 단순 성능 개선인지, 구조를 바꿀 새 아이디어인지 판단 가능. 어떤 기술은 최종 표준이 아니더라도 다음 방향을 제시한 ‘중요한 중간 단계’가 될 수 있음.

‘정답이 없다 = 기회가 있다’

  • 구조가 정해진 분야는 검증된 기술을 빠르게 익히는 능력이 중요하지만, 피지컬 AI처럼 방향이 계속 변하는 분야는 아직 중요하게 다뤄지지 않은 문제를 먼저 발견하는 능력이 큰 경쟁력.
  • 학생·연구자: 미해결 문제에서 새 연구 주제. 개발자: 자신의 AI·데이터·제어·임베디드·시뮬레이션 경험이 어디에 연결되는지 판단. 창업자: 모델 자체보다 데이터 수집·평가, 시뮬레이션 환경, 로봇 운영·지속 학습의 빈틈에서 제품·서비스 기회.

저자의 책 집필 관점

  • 특정 모델·프레임워크 사용법보다 긴 시간축의 흐름에 초점(출간 후 금세 구식이 되지 않도록). 2017년 트랜스포머→멀티모달 AI→ChatGPT로 지식노동 변화 체감→“이 지능을 로봇과 연결할 수 없나?”라는 질문이 피지컬 AI로 이어진 계보를 따라감.
  • 구성: LLM 플래닝→VLA 아키텍처 발전 / 중간표현·연속 액션·명시적 추론·계층형 시스템 / 데이터 수집·가공, 시뮬레이션·합성데이터, 파인튜닝·온디바이스 배포, 지속 학습, 주요 연구그룹·빅테크 프로젝트.
  • 전달하려는 것: 하나의 정답이 아니라 새 기술을 바라보는 기준과 방향감 — “이 기술이 어떤 문제를 풀고 전체 시스템에서 어떤 역할인지 스스로 판단할 수 있으면 빠른 변화 속에서도 방향을 잃지 않는다.”
